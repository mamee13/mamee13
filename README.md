## 🚀 Selected Projects

**The Automaton Auditor** — Hierarchical LangGraph multi-agent swarm for automated code governance. Three parallel detective agents (AST, RAG-lite PDF, multimodal vision) feed a dialectical judicial bench (Prosecutor, Defense, Tech Lead) with a Chief Justice conflict resolver. Full LangSmith trace observability.

**Document Intelligence Refinery** — Five-stage agentic pipeline for enterprise document extraction with confidence-gated escalation (fast text → layout-aware → VLM), semantic chunking, hierarchical PageIndex navigation, and full provenance chains (page + bounding box citations).

**The Brownfield Cartographer** — Multi-agent codebase intelligence system. Surveyor (tree-sitter + PageRank on import graphs), Hydrologist (sqlglot data lineage DAG), Semanticist (LLM purpose extraction + k-means clustering), Archivist (auto-generated living CODEBASE.md). LangGraph Navigator with blast radius analysis.

**Agentic Event Store & Audit Infrastructure** — Production PostgreSQL event sourcing system: append-only store, optimistic concurrency, CQRS projections, cryptographic hash chains for tamper detection, outbox pattern, MCP-compatible API, and Gas Town persistent ledger for agent memory recovery.

**The Oracle Forge (Data Agent)** — NL-to-SQL analytics agent benchmarked against UC Berkeley's DataAgentBench. Three-layer context architecture, multi-database routing (PostgreSQL/MongoDB/SQLite/DuckDB), self-correcting execution with failure diagnosis.

**Conversion Engine for Sales Automation** — B2B lead generation pipeline with signal enrichment (Crunchbase, job-post velocity, layoffs.fyi, AI maturity scoring), multi-channel outreach (email + SMS), HubSpot CRM via MCP, Cal.com booking. Benchmarked against τ²-Bench.

**Sales Agent Evaluation Bench (Tenacious-Bench v0.1)** — 200–300 task evaluation dataset built via four authoring modes (trace-derived, programmatic sweeps, Magpie-style multi-LLM synthesis, adversarial hand-authored). LLM-as-a-judge filtering, contamination checks, inter-rater agreement. LoRA adapter trained on Qwen 3.5 via Unsloth, published to HuggingFace Hub.

**Bota Parking System** — Full smart parking ecosystem: Driver App, Watchman App, Admin Panel — unified Node.js/PostgreSQL backend with real-time availability, QR-based entry/exit, and integrated payments. Three production mobile apps.

**Clinic CRM (Nexa Business Group)** — Flutter CRM serving 8+ dental clinic branches. Clean architecture, GetX/Dio, Node.js/PostgreSQL backend, published to Google Play Store. Multi-clinic patient management, scheduling, JWT + RBAC, Cloudinary media.
